Exceptions in Python

In Python, exceptions are errors that occur during the execution of a program, interrupting its normal flow. They provide a way to handle unexpected situations gracefully without crashing the entire program.

Types of Exceptions

Python has several built-in exceptions, and users can also create custom exceptions. Common exceptions include:

  • Zero division error: Raised when division or modulo by zero occurs.
  • Type error: Raised when an operation or function is applied to an object of an inappropriate type.
  • value error: Raised when a function receives an argument of the right type but with an inappropriate value.
  • File Not Found Error: Raised when a file operation (like opening a file) fails because the file doesn't exist.

Explanation:

  • func:The function being decorated.
  • wrapper: A new function that adds extra functionality before and/or after the original function is executed.

Applying a decorator:

You can apply a decorator to a function by placing the @decorator_name directly above the function definition:

Syntax:

@decorator
def my_function():
    print("This is my function.")
my_function()

When my_function() is called, the decorator modifies its behavior by adding code before or after the original function's execution.

Example:


def greet_decorator(func):
    def wrapper():
        print("Hello!")
        func()
        print("Goodbye!")
    return wrapper

@greet_decorator
def say_hello():
    print("How are you?")

say_hello()

Output:

Hello!
How are you?
Goodbye!

In this example, the greet_decorator adds the "Hello!" and "Goodbye!" messages around the say_hello() function.
